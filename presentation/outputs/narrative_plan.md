# План презентации

## Аудитория

Партнер, инвестор или потенциальный клиент, которому нужно быстро понять ценность локального интерактивного прототипа платежной платформы.

## Цель

Показать, что решение не является статичным макетом: в нем есть роли, модули, мок-данные, локальная база, API routes, сценарии, статусы, балансы, апелляции, уведомления и журнал событий.

## Нарратив

1. У продукта есть понятная бизнес-задача: объяснить и проверить платежный контур без реальных денег.
2. Прототип покрывает ключевые роли и бизнес-модули.
3. Операции проходят жизненный цикл, который влияет на баланс и риск.
4. Апелляции, уведомления и журнал событий делают систему объяснимой.
5. Решение можно запускать локально, показывать партнеру и использовать как базу для следующего этапа.

## Слайды

1. Обложка и позиционирование.
2. Что уже собрано в решении.
3. Роли и зоны ответственности.
4. Модули продукта.
5. Жизненный цикл платежного ордера.
6. Финансовая логика: балансы, комиссии, холды.
7. Апелляции и управление спорными операциями.
8. API-демо и интеграции.
9. Интерактивные сценарии для демонстрации.
10. Ценность для партнера/инвестора и следующий этап.

## Визуальная система

Премиальный B2B fintech/SaaS: теплый фон, темно-зеленая основа, jade-акцент, brass-акцент для финансовых элементов, карточки, бейджи, аккуратные схемы и много воздуха.

## План editability

Все важные тексты, карточки, схемы, подписи и блоки создаются как редактируемые объекты PowerPoint. Презентация должна быть пригодна для правки вручную.
